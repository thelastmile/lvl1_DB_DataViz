Introduction to Databases and Data Visualization

Welcome to the next learning module of the class. This segment will introduce two closely related topics that will become important building blocks in the development of your coding skills, databases and data visualization.

Databases

What is data? Simply put, data is a set of individual pieces of information. Here is a link to a short article on data from Wikipedia. So if data is a collection of individual bits, a dataset is a grouping of data pertaining to a particular subject.

As you might imagine, datasets can get pretty large. Modern statistical and scientific analysis commonly works with datasets numbering well into the billions of elements. Obviously we need a way to organize, sort, filter and manipulate these datasets if we're going to get any work done. This is where a database comes in.

A database is a system, typically software-based, that provides exactly this type of functionality. There are a number of different types of databases including:

  • Relational Databases
  • Object-oriented or NoSQL Databases
  • Graph Databases

Each of these databases are optimized for handling data in different ways, but at their core they all provide the common function of data management. An most obvious difference between databases is the language and syntax used to interact with them, and one of the most widely used formats is Structured Query Language, or SQL.

Data Visualization

The field of data visualization offers a great opportunity to combine technical and design skills to create meaningful visual representations of data. It's often the case that the complexity and volume of a dataset make it simply overwhelming to assimilate through columns of numbers and text. Data Visualization is the art and science of finding the best way to translate complex datasets into meaningful graphics. This enables better communication and understanding of complex data, and if it's done well can lead to interpretations that might not have been possible otherwise.

Data Visualization is not a product of computer science. Graphic representation of data has been around for hundreds of years. One of the best data visualizations ever was created by the 19th century French engineer Charles Joseph Minard, and depicted Napoleon's disastrous 1812 military campaign.

As you can see from the linked article, Minard was able to use visual terms to clearly and accurately relate information about numerous data points related to Napoleon's march.

D3.js

For this module, you will be working with a popular library called D3.js. Many data visualizations created for the web are based on D3, and it provides the basis for numerous other libraries that extend it's functionality. D3 is a very powerful library, and it leverages native HTML5 DOM interaction and SVG (Scalable Vector Graphics) capabilities.
